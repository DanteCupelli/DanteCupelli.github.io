Manual Analysis With CLI

A tool or approach towards this problem was to see what we could learn from manually combing through the data of the phone using a hexdump software like ghex. Although this method is long and can be extremely tedious it would be essential if there was some hidden data in a host protected area or possible data hidden within another file. Using the mmls command we can see the number of partitions on the extracted image.

Figure 2.0: ‘MMLS’ of the mobile device

The most interesting partition on this device is the userdata location. Using the DD command we can just take the last two partitions and do some file system analysis. Using the fsstat command we can see what type of file system we are dealing with.

Figure 2.1: ‘FSSTAT’ of the mobile device

This partition is a Linux based OS which makes sense due to it being an android device. We also now know that it is specifically an EXT4 file system. This will help us when we start to analyze the raw hex data in ghex. To know where data is we need to understand the file system we will be looking into. For this extremely basic forensic activity, we will mostly be doing recon to find out if there is any actual user data on the device.

There are some ways of effectively looking for data in hex dumps and one of those ways is to search the image for popular file types to find user data. For example, in this case, we were able to search the image for mp3 file extensions and found one of the user's music files. Now what this provides to us in terms of investigation is not much but we can at least confirm that there is data still on the device.

Figure 2.2: MP3 file on the mobile device

Now that we confirmed that there is some data on the device It would be best to continue to scrape along looking for identifiable or personal information. We were able to find a personal email on the hex dump shown below. This metadata shows the sender's and recipient's email as well as the subject line. However, I believe due to the encryption built into email protocols we are not able to see the full message. Although we were able to find some identifiable data just from a hex dump it became increasingly clear that this is not the most efficient way to search the phone for data. There are many different types of software that do this kind of manual work for us. It also became clear that a lot of this data is not going to be readable in ASCII.

Figure 2.4: Metadata on Mobile Device
