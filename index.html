Android Forensics

Executive Summary

April 5th, 2020

With the widespread utilization and integration of Smartphones in our daily lives, there is undoubtedly important in mobile device forensics. In addition to the increased usage, a publically available Android distribution dashboard informs us that there is a large percentage of phones using heavily outdated platform distributions.

We conducted a live analysis on a Samsung Galaxy S5 device and created an image of the disk to be distributed amongst each investigator. With the resource available to everyone, we chose two forms of manual analysis (GUI and CLI) to conduct our experiments. We then compared the results against the widely popular Autopsy tool (A part of the Sleuth Kit).

After much testing we were able to uncover the items placed on the mobile device through all methods, however the best tool to find all these items was the Autopsy tool.

Conclusion

To start our project we were wondering just how easy it is to get a disk image of an old android device and find all the information on it. Using certain dev tools built into the android shell we were able to quite simply pull the image off of the device and begin our analysis on a file system neither of the investigators knew about. The decision on the software was looking at basic tools for beginners to get a good start on finding information on the device. Naturally, it makes sense to compare the two software for what they are capable of.

Autopsy and Data FTK are similar as both options present a nice readable GUI for the user to look at data on the disk, and both are much more efficient and useful compared to manual analysis. The only time the manual analysis should be used is if the investigator is aware that there is hidden data on the drive. The main division in capabilities between the two software was the ability to view the DB files. Autopsy has android forensics built into the tool kit while FTK is more a base software for viewing disks. This allowed the software to view the SQL tables without having any real knowledge of SQL or requiring a 3rd party software. For these reasons, we would recommend Autopsy for disk analysis out of the 2 tools however that is not to say the other methods did not keep results and could come down to preference and what the investigator is comfortable with.
