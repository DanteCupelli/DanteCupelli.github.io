Android Forensics

Executive Summary

April 5th, 2020

With the widespread utilization and integration of Smartphones in our daily lives, there is undoubtedly important in mobile device forensics. In addition to the increased usage, a publically available Android distribution dashboard informs us that there is a large percentage of phones using heavily outdated platform distributions.

We conducted a live analysis on a Samsung Galaxy S5 device and created an image of the disk to be distributed amongst each investigator. With the resource available to everyone, we chose two forms of manual analysis (GUI and CLI) to conduct our experiments. We then compared the results against the widely popular Autopsy tool (A part of the Sleuth Kit).

After much testing we were able to uncover the items placed on the mobile device through all methods, however the best tool to find all these items was the Autopsy tool. 

Setting Up the Environment

With user access, the main method of obtaining our mobile device image is to use adb to perform CLI commands using ‘adb.exe’. Once the mobile device is connected to the forensics machine, it will be recognized as a device with a subsequent ID. This can be accomplished using the adb.exe devices command. Now that the device is recognized, we obtain shell access by adb.exe -d shell.

Figure 1.0: Android Debug Bridge and phone ID.


A command like cat /proc/partitions can be used to view and determine which partition we can utilize. For our image and other forensic investigators to analyze the phone, the disk is required. In most cases, mmcblk0 will be the internal storage disk. 

  • Command to view MMC Partitions ls -l /dev/block/platform/msm_sdcc.1/by-name/ 
  • Command to view EMC Devices cat /proc/emmc
